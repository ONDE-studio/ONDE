# Инструкция по Назначению и Управлению Ролью Администратора (Admin Bootstrap & Recovery)

Документ описывает процедуру безопасного одноразового назначения первого администратора студии ONDE и управление ролями без прямого редактирования SQL.

---

## 🔒 Принципы Безопасности

1. **Отсутствие хардкода Email в коде**: Email сам по себе не дает прав администратора. Права хранятся в базе данных `profiles.role` и `auth.users.app_metadata.role`.
2. **Защита от подмены при регистрации**: Пользователь **никогда** не может назначить себе роль `admin` через метаданные при регистрации (`raw_user_meta_data`). Все новые аккаунты создаются исключительно со статусом `customer`.
3. **Одноразовый Первичный Вход (One-Time Bootstrap)**: Маршрут `/admin/setup` активен **только до создания первого администратора**. Как только в системе появляется хотя бы один администратор, первичная настройка блокируется автоматически.
4. **Защита от Перебора (Rate Limiting)**: Попытки входа на страницу первоначальной настройки ограничены лимитом 5 попыток за 15 минут с одного IP.
5. **Сравнение за Постоянное Время**: Секрет настройки проверяется через `crypto.timingSafeEqual` для защиты от атак по времени (timing attacks).

## 🛠️ Шаг 1. Конфигурация Переменных Окружения

Для выполнения первоначальной настройки необходимы 2 переменные окружения на сервере:

- `OWNER_EMAIL` — Email владельца (например, `owner@onde-studio.ru`).
- `ADMIN_SETUP_SECRET` — Случайный секретный ключ высокой энтропии.

### Генерация `ADMIN_SETUP_SECRET`

Выполните команду в терминале для генерации 64-символьного hex ключа:

```bash
node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
```

### Локально (`.env.local`)
Добавьте в ваш `.env.local`:

```env
OWNER_EMAIL=ваш-email@example.com
ADMIN_SETUP_SECRET=сгенерированный-ключ-64-символа
```

### На Продакшене (Vercel)
В панели управления Vercel: **Settings ➔ Environment Variables**:
1. `OWNER_EMAIL` = `ваш-email@example.com`
2. `ADMIN_SETUP_SECRET` = `сгенерированный-ключ-64-символа`
3. Нажмите **Save** и выполните **Redeploy**.

---

## 🚀 Шаг 2. Первоначальная Настройка Администратора (Web Setup)

1. **Зарегистрируйтесь или войдите** на сайте с адресом, указанным в `OWNER_EMAIL`.
2. Откройте в браузере защищённый раздел:
   ```text
   https://onde-studio.ru/admin/setup
   ```
3. Введите сгенерированный секрет `ADMIN_SETUP_SECRET`.
4. Нажмите кнопку **«Назначить администратором»**.
5. Нажмите кнопку **«Перезайти в аккаунт»** (или выберите Выход и войти заново), чтобы обновить токен сессии.
6. Откройте `/admin` — панель управления полностью доступна.

> ⚠️ После первого назначения повторный вход на `/admin/setup` заблокирован (вернёт ошибку «Первоначальная настройка уже завершена»).

## 🛠️ Шаг 3. Управление Ролями через CLI (Recovery Script)

Для восстановления доступа или предоставления прав второму менеджеру используйте локальный CLI-скрипт.

### Посмотреть список всех пользователей и их ролей:
```bash
npm run admin:list
```

### Предоставить роль администратора:
```bash
npm run admin:grant -- user@example.com
```

### Отозвать роль администратора:
```bash
npm run admin:revoke -- user@example.com
```

> 🛡️ **Защита от блокировки**: Скрипт не позволит отозвать роль у единственного остающегося администратора в системе без явного флага `--force`:
> ```bash
> npm run admin:revoke -- last-admin@example.com --force
> ```

## 📝 Журнал Аудита (Audit Log)

Каждая операция изменения роли или попытка первичной настройки фиксируется в таблице `admin_audit_log`:

- `admin_bootstrap_succeeded`
- `admin_bootstrap_failed`
- `admin_role_granted_cli`
- `admin_role_revoked_cli`

Просмотреть журнал аудита может только администратор через Supabase SQL Editor:
```sql
SELECT * FROM public.admin_audit_log ORDER BY created_at DESC;
```
